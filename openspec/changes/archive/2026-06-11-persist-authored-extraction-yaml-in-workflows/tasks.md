# Persist Authored Extraction YAML In Workflows Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:test-driven-development` for each code task and `superpowers:verification-before-completion` before closeout. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Make the SDK preserve authored extraction YAML metadata in the workflow extract payload that downstream systems download and prepare again.

**Architecture:** Keep execution-only workflow groups separate from the persisted authored extraction contract. Add tests first to prove the current drop, then add `PreparedExtractionYaml.persisted_workflow_extract`, `PromptManager.persisted_workflow_extract_dict()`, and mapping input support for the SDK preparation path while keeping legacy execution methods stable.

**Tech Stack:** Python, PyYAML, Pydantic, pytest, pyright/mypy where configured.
